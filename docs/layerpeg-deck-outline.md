# LayerPeg / NormRef 路演提纲（10页）

## Slide 1：问题与机会
- 工程现场的核心问题：规范难执行、责任难闭环、版本难统一。
- 机会：把规范从 PDF 变成可执行协议。

## Slide 2：一句话方案
- LayerPeg：执行协议层。
- NormRef：产品与服务层。
- 核心口号：不是“查规范”，是“执行规范”。

## Slide 3：核心闭环
- 输入 -> 解析 -> Path/Gate/State -> Proof -> 输出 -> 回写。
- AI 仅负责理解与表达，不负责最终裁决。

## Slide 4：架构总览
- 五层产品视图。
- 三层因果执行视图。
- 文档五层协议（Header/Gate/Body/Proof/State）。

## Slide 5：差异化能力
- `v:// + ProjectUTXO` 主权寻址。
- `Fork/Split/Merge` 工程版本操作。
- Mapping API：桩号 -> 一切。

## Slide 6：产品矩阵
- PegBot、FormPeg、MarkUnit、PegUnit。
- GUI + CLI + API 三入口协同。

## Slide 7：可执行交付
- SpecBundle：`md + json + hash`。
- Postman + TypeScript SDK 可即刻对接。

## Slide 8：业务价值与指标
- 响应速度、强条准确率、清单采纳率、违规拦截。
- 审计可回放、责任可签名。

## Slide 9：商业模式
- SaaS订阅 + API计费 + 协议/认证服务 + 实施交付。
- 先工程深耕，再跨行业扩展（DocBot）。

## Slide 10：路线图与请求
- 0-3个月：MVP闭环。
- 3-12个月：桥隧扩展与生态接入。
- 需要的资源：试点项目、标准合作、行业渠道。
