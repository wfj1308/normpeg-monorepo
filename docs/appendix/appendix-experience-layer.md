# 附录D：产品体验层（MarkUnit × PegUnit × PegBot）

## 范围
来源于历史章节：29、30、31。

## D1. 关系定位
- MarkUnit：主工作台（编辑与协作）。
- PegUnit：案例与模板资源层。
- PegBot：规范解析与合规助手。

## D2. 推荐交互
- 主界面写作不中断。
- 右侧上下文卡片：条文/案例/动作。
- 一键 Fork 案例并插入引用。
- 关键动作进入 CLI 批量执行或 CI 自动检查。

## D3. CLI 核心命令
- `ask`、`lookup`、`checklist`、`fork`、`diff`
- `status`、`commit`、`log`、`sync`

## D4. 差异化
- 不是“查规范”，而是“执行规范”。
- Gate 拦截 + Proof 存证 + 多方签名构成闭环。
