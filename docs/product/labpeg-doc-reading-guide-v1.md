# LabPeg 文档导航（V1）

## 1. 先看这个（3 分钟）

如果你只想快速进入状态，按这个顺序读：

1. `labpeg-business-development-requirements-v1.md`
2. `labpeg-web-mobile-prototype-interaction-checklist-v1.md`
3. `labpeg-frontend-development-task-breakdown-v1.md`

## 2. 每份文档负责什么

| 文档 | 作用 | 适合谁 |
|---|---|---|
| `labpeg-business-development-requirements-v1.md` | 业务需求与验收基线（What） | 产品、后端、测试、实施 |
| `labpeg-web-mobile-prototype-interaction-checklist-v1.md` | 页面原型与交互规则（How UI works） | 产品、UI、前端、测试 |
| `labpeg-frontend-development-task-breakdown-v1.md` | 前端任务拆分与排期（Execution） | 前端负责人、研发经理、PM |

## 3. 什么时候看哪份

1. 需求评审：看主需求文档。
2. 原型评审：看原型交互清单。
3. 开发排期：看前端任务拆分。
4. 联调验收：主需求 + 交互清单一起看。

## 4. 版本管理建议

1. 主需求变更时，先改主需求文档。
2. 页面行为变化时，再改交互清单。
3. 最后同步任务拆分，避免三份文档不同步。
