# 高速公路全标准解析实施方案（NormDoc + Gate）v1.1

版本日期：2026-04-15  
适用范围：LabPeg / NormRef / LayerPeg Gate / FormPeg 联动

## 1. 目标

按“先保当前项目、再扩全标准”的策略，建立可持续解析流水线：

1. 将标准出版社采购的 PDF 原文转为可执行 NormDoc。
2. 输出统一交付物：`.normdoc`（规范对象包）+ 校验报告 + 专家签名。
3. 支持版本血缘、增量修订、项目覆盖，不改原文只叠加补丁。
4. 让 FormPeg 能直接调用 Gate 做实时校验与阻断。

## 3. 输入与输出契约

## 3.1 输入

- 标准 PDF 原文（标准出版社购买版，带版本信息）。

## 3.2 输出

- `.normdoc` 文件包（建议 zip 包，内含 `normdoc.json` + `normdoc.md` + `meta.json`）
- 校验报告（schema 校验、规则完整性、冲突检测）
- 专家签名记录（审核结论、签名人、签名时间、证据链）

## 4. 解析生产流程（固定三步）

```text
输入：PDF规范原文
  -> Step 1 AI预处理（大模型）
  -> Step 2 人工校验（领域专家）
  -> Step 3 NormDoc生成（自动化）
  -> 输出：.normdoc + 校验报告 + 专家签名
```

## 4.1 Step 1：AI预处理（大模型）

1. 章节结构识别：构建条款树。
2. 表格自动提取：实测项目结构化。
3. 公式识别：表达式标准化（含 LaTeX 映射）。
4. 术语抽取：生成 CMap 候选。

当前技术映射：`OpenDataLoader + adapter` 负责 PDF 到 `odl_json/odl_markdown/schema`。

## 4.2 Step 2：人工校验（领域专家）

1. 关键条款确认（安全、强条、红线项）。
2. 表格字段校准（单位、精度、范围、口径）。
3. 规则逻辑验证（if-then 条件与边界）。
4. 交叉引用标注（规范间关联与冲突裁决）。

当前状态：该环节已定义，需建设“专家审核台”作为发布前强制门禁。

## 4.3 Step 3：NormDoc生成（自动化）

1. 按模板填充对象：`Header/Body/Gate/Trailer/IncrementalUpdates`。
2. 计算字段公式编译与依赖图构建。
3. 校验规则索引构建（Gate Table Rule Tree）。
4. 版本血缘标注（replaces/patch/override）。

当前技术映射：

- 已接入模板：`T0921`、`T0931`、`T0951`
- 待补模板：P1/P2 高频表

## 6. 质量与发布门禁

1. Schema 校验通过率：发布前 `100%`。
2. 关键规则误报率：<= `3%`（专家抽检口径）。
3. 解析超时率：<= `5%`（按文档分型监控）。
4. 关键条款必须有专家签名，不允许自动直发。
